Code Execution

JavaScript code is executed in two phases.

  • Memory allocation phase
  • Code execution phase

Memory Allocation Phase

Functions and variables get stored in a memory as a key value pair for execution context.

  • Assigns undefined value to the variables.
  • Copies whole function in memory.

Code Execution Phase

JavaScript executes code line by line.One line at once.It follows synchronus approach.

In the Code Execution Phase, JavaScript being a single thread language runs through the code line by line and updates the values of function and variables which are stored in the Memory Allocation Phase in the Memory Component.

Call Stack

JavaScript pushes the whole program as global context into a stack known as Call Stack.

When code execution gets completed, Global execution context gets deleted and popped out from call stack making the call stack empty.

Difference between undefined and not defined

  • undefined
  1. When a variable is declared but no value is assigned to it.
  2. Variable is called for execution.
  3. For example-

    var x; console.log(x)

    Output of code will be undefined.

  • Notdefined
  1. When a variable is called for execution before declaring it.
  2. For example-

    console.log(y);

    y=7;

    Output of code will be notdefined.

  3. if a variable is declared using var keyword than value of variable undefined will be printed.
